Yahtzee SVG Rendering Examples

These examples demonstrate the SVG rendering capabilities of the Yahtzee plugin for the Async Boardgame Service.

1. New Game - First Roll

A fresh game showing the initial dice roll with no categories filled yet. All scorecard categories are available (shown with reduced opacity).

Features Demonstrated:

  • Clean dice rendering with dot patterns (3, 1, 4, 1, 5)
  • Empty scorecard with all 13 categories available
  • Current player indication (blue border around scorecard)
  • Game phase information (Roll 1 of 3)
  • Action guidance text

2. Mid-Turn with Kept Dice

Shows a player on their second roll with some dice kept (green borders) and some categories already filled in the scorecard.

Features Demonstrated:

  • Keep indicators on dice (green borders around kept dice)
  • Partially filled scorecard (Ones: 2, Twos: 6, Full House: 25)
  • Bold text for filled categories vs. faded text for available ones
  • Running totals (Upper Total: 8, Grand Total: 33)
  • Roll count indication (Roll 2 of 3)

3. Scoring Phase - Yahtzee Roll!

A lucky player has rolled a Yahtzee (five 6s) and is in the scoring phase, ready to choose a category.

Features Demonstrated:

  • All dice showing the same value (Yahtzee roll)
  • All dice marked as kept (green borders)
  • Scoring phase indication
  • Partially filled scorecard showing game progress
  • Upper section bonus eligibility tracking

4. Two-Player Game

A two-player game in progress showing both players' scorecards side by side with the current player highlighted.

Features Demonstrated:

  • Multiple player scorecards displayed side by side
  • Current player indication (Alice has the blue border)
  • Different progress levels for each player
  • Automatic layout scaling for multiple players
  • Dice with selective keep indicators

5. Completed Game with Winner

A finished game showing the final scores and winner announcement. Alice achieved the upper section bonus.

Features Demonstrated:

  • Completed scorecards with all categories filled
  • Upper section bonus calculation (Alice: 84 points + 35 bonus)
  • Final grand totals (Alice: 392, Bob: 122)
  • Winner announcement in game info area
  • Game completion state rendering

6. Four-Player Game

A four-player game demonstrating the layout scaling and current player indication with multiple participants.

Features Demonstrated:

  • Four player scorecards with automatic horizontal layout
  • Current player indication (Carol is highlighted)
  • Varied scorecard progress across players
  • Scalable viewBox dimensions
  • Dice with three of a kind (three 1s kept)

Technical Implementation

These SVG examples are generated by the Yahtzee renderer module, which:

  • Renders dice with accurate dot patterns and visual keep indicators
  • Displays scorecards with all 13 categories, totals, and bonus calculations
  • Indicates current player with visual highlighting
  • Shows game phase information and available actions
  • Scales automatically for 1-8 players with proper layout
  • Uses layered rendering with proper z-index ordering
  • Provides complete SVG compatible with web browsers and vector graphics tools
